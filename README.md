# 🚀 Space Tour Reservation System

A simple reservation application built to learn AWS services, serverless architecture, and how applications can be deployed on AWS.

The application allows users to complete a multi-step space tour reservation process, receive a reservation ID, and get an email confirmation after the reservation is successfully created.

## What I Learned

* Building a project teaches much more than reading about services individually.
* There is no perfect architecture. Different applications can have different solutions.
* Before designing an architecture, it is important to understand the problem, requirements, expected workload, and future needs.
* Security, monitoring, troubleshooting, and cost should be considered from the beginning, not added later.
* Serverless services reduce infrastructure management, but they also introduce service limits, integrations, and debugging challenges.
* Asynchronous workflows helped me understand how application components can work independently.
* Automating deployments with GitHub Actions made updates faster and removed many manual deployment steps.
* Small architectural decisions can affect scalability, maintainability, and operational effort.
* This project improved the way I think about designing, deploying, and operating cloud applications.
